# សំណួរត្រៀមសម្រាប់ឆ្លើយលោកគ្រូពេលធ្វើបទបង្ហាញ (Teacher Q&A)

នេះគឺជាចម្លើយដែលរៀបចំយ៉ាងពិសេសសម្រាប់តបទៅនឹងសំណួររបស់លោកគ្រូ។ ការឆ្លើយបែបនេះបង្ហាញថាក្រុមរបស់យើងមានចំណេះដឹងច្បាស់លាស់ និងប្រើប្រាស់ស្តង់ដារការងាររបស់ក្រុមហ៊ុនបច្ចេកវិទ្យាទំនើប។

---

### ❓ សំណួរទី១៖ "ហេតុអ្វីបានជាក្រុមអ្នកមិនប្រើ MySQL (ឬ XAMPP/phpMyAdmin)?"

**💡 របៀបឆ្លើយប្រាប់គ្រូ៖**
> "លោកគ្រូ តាមពិតទៅប្រព័ន្ធរបស់យើង **កំពុងតែប្រើប្រាស់ MySQL** ជា Database ធម្មតា! ប៉ុន្តែជំនួសឱ្យការប្រើ XAMPP ដំណើរការ Database នៅលើម៉ាស៊ីន Local ដែលយឺត និងពិបាកឱ្យសមាជិកក្រុមតភ្ជាប់ចូល យើងបានលើក Database នោះទៅដាក់នៅលើ Cloud ដោយប្រើប្រាស់សេវាកម្ម **Aiven Cloud MySQL**។ 
>
> អ្វីដែលយើងប្តូរ គឺកម្មវិធីសម្រាប់មើល Database (Viewer)។ ជំនួសឱ្យការប្រើ phpMyAdmin យើងងាកមកប្រើកម្មវិធី **TablePlus** វិញ ព្រោះ Cloud Database ត្រូវការប្រព័ន្ធសុវត្ថិភាពខ្ពស់ដោយប្រើ SSL Certificate (`ca.pem`) ហើយកម្មវិធី TablePlus អាចភ្ជាប់ជាមួយ Cloud ទំនើបៗបានល្អ សុវត្ថិភាពខ្ពស់ និងដើរលឿនជាង phpMyAdmin។"

---

### ❓ សំណួរទី២៖ "បើមិនប្រើ phpMyAdmin តើធ្វើដូចម្តេចទើបអាចបង្កើត Table បាន?"

**💡 របៀបឆ្លើយប្រាប់គ្រូ៖**
> "ក្នុងការសរសេរកម្មវិធីតាមស្តង់ដារទំនើប គេលែងបង្កើត Table ដោយការចុច Right-Click បង្កើតដោយដៃ (Manually) ទៀតហើយលោកគ្រូ។ ជំនួសមកវិញ ក្រុមយើងខ្ញុំប្រើប្រាស់ប្រព័ន្ធ **Laravel Migrations**។
>
> **Migration** គឺប្រៀបដូចជា Git សម្រាប់ Database អញ្ចឹង! យើងសរសេរកូដ PHP ដើម្បីកំណត់ថាតារាងនីមួយៗមាន Column អ្វីខ្លះ។ ពេលយើងចង់បង្កើតវា យើងគ្រាន់តែវាយបញ្ជា `php artisan migrate` នៅក្នុង Terminal នោះប្រព័ន្ធនឹងបង្កើត Table ទាំងអស់ដោយស្វ័យប្រវត្តិ។
> 
> ការធ្វើបែបនេះមានអត្ថប្រយោជន៍ខ្លាំងណាស់ ព្រោះបើសិនជាសមាជិកផ្សេងទាញយកកូដទៅកុំព្យូទ័រថ្មី គាត់មិនចាំបាច់អង្គុយបង្កើត Table ម្តងមួយៗនោះទេ គ្រាន់តែវាយបញ្ជាមួយជួរ គឺ Database ទាំងមូលនឹងលោតចេញមកយ៉ាងល្អឥតខ្ចោះ។"
