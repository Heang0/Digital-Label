# ឯកសារជំនួយសម្រាប់ការធ្វើបទបង្ហាញ (Digital Label Project)

សួស្តីសមាជិកក្រុមទាំង ៧នាក់! នេះគឺជាឯកសារដែលរៀបចំឡើងយ៉ាងពិសេស ដើម្បីជួយឱ្យអ្នកទាំងអស់គ្នាអាចធ្វើបទបង្ហាញបានយ៉ាងល្អ និងមានទំនុកចិត្តនៅចំពោះមុខគ្រូ។ ដោយសារក្រុមយើងមានសមាជិក ៧នាក់ ដូច្នេះយើងបានបែងចែកការធ្វើបទបង្ហាញជា ៧ ផ្នែក ដើម្បីឱ្យសមាជិកម្នាក់ៗមានចំណុចសំខាន់ៗរៀងៗខ្លួនដើម្បីនិយាយ។

---

## 🙋‍♀️ ទី១៖ ការណែនាំ និងរចនាសម្ព័ន្ធទូទៅនៃប្រព័ន្ធ (System Overview)
**(អ្នកទី១ គួរនិយាយចំណុចនេះ)**

* **អ្វីដែលត្រូវនិយាយ:** "ជម្រាបសួរលោកគ្រូ! ថ្ងៃនេះក្រុមយើងខ្ញុំសូមបង្ហាញពីគម្រោង **Digital Label**។ ប្រព័ន្ធរបស់យើងត្រូវបានសាងសង់ឡើងជាទម្រង់ **Modern Cloud Application** ដោយបែងចែកជា ៣ ផ្នែកធំៗដាច់ពីគ្នា (Decoupled Architecture)៖
  1. **Frontend (Next.js / React)**: សម្រាប់ផ្នែក User Interface ផ្ទុកនៅលើ **Vercel**។
  2. **Backend API (Laravel / PHP)**: ជាខួរក្បាលនៃប្រព័ន្ធ សម្រាប់គ្រប់គ្រងទិន្នន័យ ដែលផ្ទុកនៅលើ **Render** ដោយប្រើ Docker។
  3. **Database (MySQL)**: ទិន្នន័យរបស់យើងត្រូវបានរក្សាទុកយ៉ាងសុវត្ថិភាពនៅលើ Cloud Database ឈ្មោះថា **Aiven**។
* **មូលហេតុដែលបែងចែក (Why decouple?):** "ការធ្វើបែបនេះជួយឱ្យប្រព័ន្ធមានដំណើរការលឿន សុវត្ថិភាពខ្ពស់ និងងាយស្រួលក្នុងការបង្កើត Mobile App នៅថ្ងៃអនាគត ដោយមិនចាំបាច់សរសេរ Backend ឡើងវិញនោះទេ។"

---

## 🎨 ទី២៖ ផ្នែកខាងមុខ និងបទពិសោធន៍អ្នកប្រើប្រាស់ (Frontend & UI)
**(អ្នកទី២ គួរនិយាយចំណុចនេះ)**

* **អ្វីដែលត្រូវនិយាយ:** "សម្រាប់ផ្នែក Frontend យើងប្រើប្រាស់ **Next.js** ដើម្បីបង្កើត UI ដែលមានភាពទាក់ទាញ និងងាយស្រួលប្រើ។ យើងបានប្រើប្រាស់បច្ចេកវិទ្យាទំនើបៗ ដើម្បីធានាថាគេហទំព័រដើរបានលឿន (Fast Rendering)។"
* **កូដដែលត្រូវបង្ហាញគ្រូ (`frontend/lib/api.ts`):** 
  *"លោកគ្រូអាចមើលឯកសារនេះ វាគឺជាកន្លែងដែល Frontend របស់យើងទាក់ទងទៅកាន់ Backend។ យើងមានកូដកណ្តាលមួយដែលយក `NEXT_PUBLIC_API_URL` មកប្រើ ដើម្បីធ្វើការ Request ទាញយកទិន្នន័យ Product ឬធ្វើការ Login ដោយសុវត្ថិភាព។"*

---

## ⚙️ ទី៣៖ ផ្នែកខាងក្រោយ និងប្រព័ន្ធសុវត្ថិភាព (Backend & Security)
**(អ្នកទី៣ គួរនិយាយចំណុចនេះ)**

* **អ្វីដែលត្រូវនិយាយ:** "សម្រាប់ Backend យើងបានជ្រើសរើស **Laravel** ព្រោះវាមានប្រព័ន្ធសុវត្ថិភាពល្អ និងរៀបចំកូដបានស្អាត។ រាល់ការទាញយកទិន្នន័យពី Frontend ត្រូវឆ្លងកាត់ការត្រួតពិនិត្យ (Authentication) យ៉ាងតឹងរ៉ឹងដើម្បីការពារកុំឱ្យមានការលួចទិន្នន័យ។"
* **ចំណុចពិសេស:** "យើងបានដាក់ Backend នេះនៅលើ **Render** ដែលវាដំណើរការដោយស្វ័យប្រវត្តិ។" 
*(បញ្ជាក់ប្រាប់គ្រូថា៖ ដោយសារយើងប្រើ Free Tier ពេលខ្លះ Backend ត្រូវចំណាយពេលបន្តិចដើម្បី 'ភ្ញាក់ពីគេង' ប្រសិនបើគ្មានអ្នកចូលប្រើលើសពី ១៥នាទី)*។

## 🗄️ ទី៤៖ ការគ្រប់គ្រងទិន្នន័យ (Database Management & Seeding)
**(អ្នកទី៤ គួរនិយាយចំណុចនេះ)**

* **អ្វីដែលត្រូវនិយាយ:** "ទិន្នន័យរបស់យើងទាំងអស់គឺស្ថិតនៅលើ **Aiven Cloud MySQL**។ ដើម្បីគ្រប់គ្រងវា យើងប្រើកម្មវិធី **TablePlus** ជំនួសឲ្យ MySQL Workbench ព្រោះវាលឿនជាង និងងាយស្រួលភ្ជាប់ជាមួយ Cloud តាមរយៈការប្រើប្រាស់ SSL Certificate (`ca.pem`) យ៉ាងមានសុវត្ថិភាព។"
* **កូដដែលត្រូវបង្ហាញគ្រូ (`backend/database/seeders/DatabaseSeeder.php`):** 
  *"ដើម្បីមានទិន្នន័យសម្រាប់តេស្ត យើងបានសរសេរឯកសារ `DatabaseSeeder.php` នេះ។ ជំនួសឱ្យការបញ្ចូលទិន្នន័យម្តងមួយៗ យើងគ្រាន់តែវាយ Command មួយ នោះ Laravel នឹងបង្កើតតារាង និងបញ្ចូលទិន្នន័យអ្នកប្រើប្រាស់ (Admin, Staff Demo) រួមទាំងផលិតផលគំរូដោយស្វ័យប្រវត្តិ។"*

---

## 🖼️ ទី៥៖ ការរក្សាទុករូបភាពនៅលើ Cloud (ImageKit Integration)
**(អ្នកទី៥ គួរនិយាយចំណុចនេះ)**

* **អ្វីដែលត្រូវនិយាយ:** "ជាធម្មតា ការរក្សាទុករូបភាពនៅលើ Server ផ្ទាល់អាចធ្វើឱ្យ Server យឺត និងពេញទំហំ។ ដូច្នេះ យើងបានប្រើប្រាស់សេវាកម្មទី៣ ឈ្មោះថា **ImageKit.io**។"
* **កូដដែលត្រូវបង្ហាញគ្រូ (`backend/app/Http/Controllers/Api/UploadController.php`):**
  *"នៅក្នុងកូដនេះ ពេលអ្នកប្រើប្រាស់ Upload រូបភាព Backend របស់យើងនឹងយកវាទៅបម្លែងជា Base64 ហើយបញ្ជូនវាទៅរក្សាទុកនៅ ImageKit ដោយប្រើ Private Key។ បន្ទាប់មក យើងគ្រាន់តែរក្សាទុក URL នៃរូបភាពនោះទៅក្នុង Database ជាការស្រេច។"*

## 🔄 ទី៦៖ ការធ្វើការជាក្រុម និងការដាក់ឱ្យដំណើរការ (Collaboration & Auto-Deployment)
**(អ្នកទី៦ គួរនិយាយចំណុចនេះ)**

* **អ្វីដែលត្រូវនិយាយ:** "ដោយសារយើងមានសមាជិកច្រើន យើងប្រើប្រាស់ **GitHub** ដើម្បីគ្រប់គ្រងកូដ។ ពួកយើងបានភ្ជាប់គណនី GitHub រួមគ្នាតែមួយ (Shared Repository)។"
* **ចំណុចពិសេស (CI/CD):** "អ្វីដែលអស្ចារ្យនោះគឺ នៅពេលដែលយើងសរសេរកូដរួច ហើយវាយបញ្ជា `git push` ប្រព័ន្ធ **Vercel** និង **Render** នឹងដឹងដោយស្វ័យប្រវត្តិ ហើយពួកវានឹងទាញយកកូដថ្មីនោះទៅ Deploy (ដាក់ឱ្យដំណើរការ) លើអនឡាញភ្លាមៗដោយយើងមិនបាច់ធ្វើអ្វីបន្ថែមទៀតនោះទេ។"
